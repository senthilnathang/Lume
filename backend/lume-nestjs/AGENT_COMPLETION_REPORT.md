## 📋 Known TypeScript Issues to Fix

### DTO Property Initializer Errors
**Pattern**: Properties missing `!` (non-null assertion) or default values
**Example**: `Property 'title' has no initializer and is not definitely assigned in the constructor`
**Fix**: Add `!` after property type: `title!: string`

**Files affected**:
- activities/dtos/create-activity.dto.ts
- advanced_features/dtos/*.ts (multiple)
- base/dtos/*.ts (multiple)
- base_automation/dtos/*.ts (multiple)

**Batch Fix Command**:
```bash
# For each DTO file, add ! to uninitialized properties
sed -i 's/^\s*\([a-zA-Z_][a-zA-Z0-9_]*\):\s*\(string\|number\|boolean\|Date\|any\)$/  \1!: \2/g' src/modules/*/dtos/*.ts
```

### Schema Import Errors
**Pattern**: `Cannot find module '../models/schema'`
**Issue**: Drizzle schema imports not found
**Files affected**:
- base_automation/services/automation.service.ts

**Fix**: Check if schema file exists at expected path, or import from correct location

### Prisma Type Mismatches
**Pattern**: `'rolePermissions' does not exist in type 'RoleInclude'` (should be `'role_permissions'`)
**Issue**: Mixing camelCase (TypeScript) with snake_case (Prisma schema)
**Files affected**:
- base_rbac/services/base-rbac.service.ts

**Fix**: Use snake_case for Prisma fields: `role_permissions` instead of `rolePermissions`

### Missing Controllers

## 🔧 TypeScript Fix Strategy

### Phase 1: DTO Fixes (All modules)
1. Add `!` to all uninitialized DTO properties
2. Ensure `@IsNotEmpty()` matches `!` marker
3. Add `@IsOptional()` for optional fields instead of `?`

**Template Fix**:
```typescript
// Before
export class CreateDto {
  @IsString()
  name: string;  // ❌ Error: no initializer
}

// After
export class CreateDto {
  @IsString()
  @IsNotEmpty()
  name!: string;  // ✅ Fixed
}
```
