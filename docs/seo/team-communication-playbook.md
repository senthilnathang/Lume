# Team Communication & Alignment Playbook: Lume v2.0 Launch

## Purpose

**Aligned teams execute launches.** This playbook provides ready-to-send templates for:
- Daily stand-ups
- Weekly syncs
- Status updates
- Escalations
- Celebrations
- Post-mortems

**Use these templates to ensure everyone knows what's happening and why.**

---

## Part 1: Daily Stand-Up Template

### Time: 9:00 AM UTC
### Duration: 15 minutes
### Participants: All 7 roles (founder, content, marketing, devops, community, support, analytics)

**Slack notification (posted in #launch at 8:55 AM):**

```
🚀 DAILY STAND-UP IN 5 MIN
📍 Zoom: [link]
⏱️ 15 min (9:00-9:15 AM UTC)

If you can't make it, update #standup-summary

Today's focus: [TOPIC - see below for daily topics]
```

### Daily Stand-Up Agenda (Same questions every day)

**9:00-9:05 (5 min): Metrics Check**
- Yesterday's visitors: ___
- Yesterday's signups: ___
- Yesterday's GitHub stars: ___
- Any anomalies? (spikes, drops, errors?)

**9:05-9:10 (5 min): Blockers & Issues**
- Is anyone blocked? (waiting on someone/something?)
- Are there critical issues? (bugs, downtime, etc.?)
- Do we need to escalate anything?

**9:10-9:15 (5 min): Today's Priorities**
- What's the #1 priority today? (unanimous agreement)
- Who's responsible for each priority?
- Any coordination needed between roles?

### Daily Focus Topics (Rotate)

| Day | Focus | Key Owner | Sample Question |
|-----|-------|-----------|---|
| **Mon** | Content & Metrics | Content Lead | Which blog posts drove most traffic? |

| **Thu** | Technical & Infrastructure | DevOps Lead | Any performance issues? Uptime solid? |
| **Fri** | Week Wrap-Up | Founder | Did we hit our weekly targets? What's next? |

## Part 5: Crisis Communication Templates

### Use When: Red Zone risk materializes

**Template 1: Infrastructure Issue (Website Down)**

```
🚨 CRITICAL: Website Infrastructure Issue

STATUS: We're aware of the issue and working on it
IMPACT: Users may experience slow load times or errors
ETA: Fix in progress, expect update in 15 minutes
UPDATES: Check this thread for updates every 5 min

What we're doing:
├─ Identified database connection pool issue
├─ Scaling up infrastructure
└─ Monitoring performance in real-time

FOLLOW THIS THREAD for updates (don't create new messages)

Question? Ask in thread 👇
```

**Template 2: Email System Failure**

```
⚠️ EMAIL DELAY NOTICED

STATUS: Email delivery slightly delayed due to volume
IMPACT: Some emails may arrive 30-60 minutes late
NEXT STEPS: 
├─ Queued emails will resend automatically
└─ We're monitoring closely

If you didn't receive signup confirmation:
→ Check spam folder first
→ Then try login directly: lume.dev/login
→ Contact support if issues persist: support@lume.dev

We apologize for the inconvenience. We're monitoring this closely.
```

**Template 3: Security or Major Bug**

```
🔒 SECURITY NOTICE - [Date/Time UTC]

We're aware of a [critical/high/medium] security issue affecting [users/feature].

WHAT HAPPENED:
[Brief, honest description of issue]

IMPACT:
[Who is affected? What can they do?]

WHAT WE'RE DOING:
1. Issue identified at [time]
2. Fix in progress (ETA: [time])
3. Deploy at [time]
4. Will notify all users when resolved

YOUR NEXT STEP:
[If needed: Reset password / Check settings / Take action]

We take security seriously. Full post-mortem and remediation plan coming soon.
Questions? Email: security@lume.dev
```

## Part 10: Post-Mortem Template

### Use When: Major incident occurs
### Timing: 24 hours after incident resolved

**Slack post + detailed document:**

```
📋 INCIDENT POST-MORTEM: [Incident Name]
Date: Sept 15, 2026

INCIDENT SUMMARY
├─ What happened: [Description]
├─ When: [Date/time UTC]
├─ Duration: [How long]
├─ Severity: [Critical/High/Medium/Low]
└─ Impact: [X users affected for Y minutes]

TIMELINE
14:30 UTC - Issue detected (form completion dropped to 5%)
14:35 UTC - Root cause identified (form had 8 fields vs 3)
14:45 UTC - Change deployed (reduced to 3 fields)
15:00 UTC - Conversion rate recovered to 2.5%

ROOT CAUSE
Form had too many fields (8 vs 3 in control), causing abandonment spike.

WHAT WE LEARNED
1. Reduce form fields by default (less friction = higher conversion)
2. Monitor form completion rate daily (not weekly)
3. Test form changes on 5% of traffic first (not 100%)

ACTIONS TO PREVENT RECURRENCE
[ ] Monitor form completion rate in daily dashboard
[ ] Implement "canary deployment" for form changes
[ ] Form design review (max 3 fields)
[ ] A/B testing framework for UX changes

OWNER: [Name]
DUE: [Date]

Full details: [Link to incident document]

Questions? Reply in thread 👇
```

## Usage Tips

✅ **Customize names and metrics** to your actual data  
✅ **Post stand-up template to Zoom chat** exactly at 9:00 AM  
✅ **Record all-hands** and post recording to Slack within 1 hour  
✅ **Use the same template every time** (consistency = clarity)  
✅ **Copy-paste templates** (don't retype)  
✅ **Celebrate wins publicly** (morale matters!)  
✅ **Address issues transparently** (teams respect honesty)  
✅ **Follow up** on action items (accountability)  

**Good communication = aligned team = successful launch** 🚀
