# Media & Visual Assets Requirements

## Website Graphics

### 1. Hero Section Image

**Specs:**
- Size: 1920x1080px (16:9 aspect ratio)
- Format: PNG or WebP (optimized)
- File size: <500KB
- Color scheme: Lume brand colors (blues, greens)

**Content:**
- Screenshot of Lume admin panel showing CRM data
- Live entities (Contacts, Companies, Opportunities)
- Visual showing entity relationships/connections
- Include hero headline overlay: "Build Your Perfect CRM Without Code"

**Variations needed:**
- Desktop (1920x1080)
- Mobile (800x600)
- Dark mode (inverse colors)

---

### 2. Feature Section Icons (4 needed)

**Specs:**
- Size: 200x200px each
- Format: SVG (scalable) or PNG
- Style: Minimalist, flat design
- Color: Lume primary brand color (#0066FF)

**Icons:**
1. **Entity Builder:** Puzzle pieces or building blocks
2. **Automation:** Lightning bolt or workflow arrows
3. **API:** Code brackets or API gateway icon
4. **Data Ownership:** Lock or shield icon

---

### 3. Comparison Table Graphics

**Specs:**
- Size: 1200x800px
- Format: PNG/WebP
- Show: Lume vs. Salesforce vs. Airtable in grid format

**Content:**
- Feature rows (pricing, users, customization, etc.)
- Color-coded checkmarks (green = Lume, yellow = limited, red = no)
- Include logos of competitors

---

### 4. Blog Post Featured Images (13 needed)

**Specs:**
- Size: 1200x630px (OG image size)
- Format: PNG/WebP
- File size: <200KB each
- Consistent style across all 13

**Templates:**
1. **Comparison posts:** Split screen (Lume vs. Competitor)
2. **How-to posts:** Step numbers, visual process
3. **Thought leadership:** Bold headline + Lume logo
4. **Case study posts:** Customer logo + metric

**Color scheme:**
- Primary: Lume brand blue
- Secondary: Light gray background
- Accent: Green for positive messages
- Text: White or dark gray

---

### 5. Social Media Graphics

**Twitter:**
- Size: 1024x512px for threads
- 4-5 graphics per thread (5 threads = 20 images)
- Format: PNG
- Include: Lume logo, key message, call-to-action

**LinkedIn:**
- Size: 1200x627px
- 6 graphics (one per article + announcement)
- Format: PNG
- Professional tone, include testimonials if applicable

**Reddit:**
- Size: 1200x627px
- 5 graphics (one per subreddit post)
- Format: PNG
- Community-appropriate style (casual, helpful)

## Video Content

### 1. Demo Video (Priority)

**Specs:**
- Duration: 2-3 minutes
- Format: MP4 (H.264 codec)
- Resolution: 1920x1080p (60fps preferred)
- File size: <100MB
- Captions: English (required)

**Content:**
1. 10s intro: "Welcome to Lume"
2. 20s setup: Deploy from GitHub
3. 30s entity creation: Add Contact, Company, Opportunity
4. 30s field configuration: Show field options
5. 20s relationships: Create links between entities
6. 20s automation: Set up trigger workflow
7. 20s API: Show REST API endpoint
8. 10s CTA: "Deploy free at lume.dev"

**Technical requirements:**
- Screen recording at 1920x1080 (no scaling)
- Smooth transitions between sections
- Background music (royalty-free, ~120BPM)
- Clear narration (professional voiceover)
- B-roll of actual Lume UI (not clickthrough)

---

### 2. "Why We Built Lume" Founder Video (Optional)

**Specs:**
- Duration: 5-7 minutes
- Format: MP4
- Resolution: 1920x1080p
- Captions: Yes

**Content:**
- Founder on camera (at desk or office)
- Background: Lume logo visible
- Topic: Why open-source, vision, problem being solved
- 3-5 customer testimonials spliced in
- Ending CTA: "Deploy free at lume.dev"

---

### 3. Feature Overview Videos (Optional, for website)

## Accessibility Requirements

**All graphics must include:**
- [ ] High contrast ratio (4.5:1 for text)
- [ ] Alt text describing images
- [ ] Captions on all videos (English minimum)
- [ ] Color not as only differentiator (use patterns/text too)
- [ ] No flickering elements (for seizure safety)

## Quality Standards

### Images
- [ ] No pixelation at 1920x1080
- [ ] File size optimized (<500KB for web)
- [ ] Consistent font (max 2 fonts per graphic)
- [ ] Brand colors accurate
- [ ] Professional quality (not DIY)

### Videos
- [ ] No compression artifacts
- [ ] Audio clear (no background noise)
- [ ] Smooth pacing (not rushed)
- [ ] Captions synchronized with audio
- [ ] Professional editing (clean transitions)

### All Assets
- [ ] Named consistently
- [ ] Properly organized in folders
- [ ] Backed up (cloud storage)
- [ ] Version control (if using design tools like Figma)
- [ ] Metadata embedded (alt text, descriptions)
