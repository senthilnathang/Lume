## Part 1: Brand Identity System

### Brand Essence
```
Mission: Empower teams to build custom CRM and database solutions 
         without coding, with full control over data and infrastructure.

Vision: Become the leading open source alternative to proprietary 
        database/CRM platforms, trusted by 10,000+ organizations.

Values:
├─ Open Source: Community-driven, transparent, no vendor lock-in
├─ Empowerment: Enable non-technical teams to build powerful apps
├─ Control: Full data ownership, self-hosted, privacy-first
├─ Simplicity: Intuitive UI, fast setup, zero coding required
└─ Reliability: Production-ready, enterprise-grade stability
```

### Brand Positioning
```
"Lume is the open source, self-hosted alternative to Airtable and 
Salesforce. Build custom CRM and database solutions without code, 
with complete control over your data."

Positioning Statement:
├─ For: Teams and organizations (5-500 people)
├─ Who want: Customizable CRM/database without vendor lock-in
├─ Lume: Provides open source, self-hosted solution
├─ Unlike: Airtable (proprietary), Salesforce (expensive, complex)
└─ We: Give complete data ownership and unlimited customization
```

### Brand Personality
```
Traits:
├─ Professional yet approachable (not stuffy)
├─ Confident about capabilities (not arrogant)
├─ Community-focused (collaborative tone)
├─ Technical but accessible (explains clearly)
└─ Trustworthy (transparent, open, honest)

Tone of Voice:
├─ Clear and direct (avoid jargon when possible)
├─ Friendly and encouraging (help users feel capable)
├─ Expert and authoritative (knowledgeable)
├─ Solution-focused (emphasize outcomes)
└─ Inclusive and welcoming (everyone can build with Lume)
```

### Typography System

**Font Family Selection**
```
Primary Font (Headlines, UI):
├─ Font: Inter (system font family)
├─ Reasoning: Modern, clean, excellent on-screen
├─ Usage: H1-H6, buttons, labels, navigation
├─ Weights: 300 (light), 400 (regular), 600 (semibold), 700 (bold)
├─ License: Open source (OFL)
└─ Fallback: -apple-system, BlinkMacSystemFont, Segoe UI

Secondary Font (Body Text):
├─ Font: System font stack (for performance)
├─ Stack: -apple-system, BlinkMacSystemFont, Segoe UI, Roboto, sans-serif
├─ Usage: Body text, paragraphs, long-form content
├─ Weight: 400 (regular), 500 (medium)
└─ Leading: 1.6 (160% line height for readability)

Code Font (Developer Docs):
├─ Font: Monaco, Menlo, Courier New, monospace
├─ Usage: Code blocks, API examples, terminal
├─ Rendering: Line numbers, syntax highlighting
└─ Size: 13px or 14px (larger than body)
```

**Typography Scale**
```
Headings:
├─ H1: 48px / 56px (headline, hero)
├─ H2: 36px / 44px (page title)
├─ H3: 28px / 36px (section title)
├─ H4: 24px / 32px (subsection)
├─ H5: 20px / 28px (small subsection)
└─ H6: 16px / 24px (smallest heading)

Body Text:
├─ Large body: 18px / 28px (intro paragraphs)
├─ Regular body: 16px / 24px (standard paragraph text)
├─ Small body: 14px / 22px (secondary text, captions)
└─ Extra small: 12px / 18px (metadata, timestamps)

UI Elements:
├─ Button text: 14px / 16px (semibold)
├─ Label text: 12px / 16px (medium)
├─ Placeholder: 14px / 16px (light gray)
└─ Helper text: 12px / 16px (secondary gray)

Line Height:
├─ Headings: 1.2 (120%)
├─ Body text: 1.6 (160%)
├─ UI elements: 1.5 (150%)
└─ Code: 1.5 (150%, monospace)

Letter Spacing:
├─ Headlines: -0.02em (tight)
├─ Body: 0em (normal)
└─ Small text: 0.01em (slightly expanded)
```
