# Lume — cPanel Deployment Guide

## Overview

Lume consists of **3 applications** deployed under a single domain. You will need to create **2 Node.js applications** in cPanel and upload **1 static folder**.

| # | Application | URL | Type | cPanel App? |
|---|------------|-----|------|-------------|
| 1 | Backend API | `ri-agri.in/backend` | Express.js | Yes — Node.js App |
| 2 | Admin Panel | `ri-agri.in/frontend` | Vue 3 SPA (static files) | No — static HTML |
| 3 | Public Website | `ri-agri.in/demo` | Nuxt 3 SSR | Yes — Node.js App |

**Deploy order: Backend first, then Admin, then Public Site.**

The backend must be running before the other two can function — the admin panel calls the backend API, and the public site fetches page content from it at render time.

---

## Prerequisites

Before you begin, ensure you have:

- [ ] cPanel hosting with **Node.js Selector** (Node.js 18 or higher)
- [ ] SSH access or cPanel Terminal
- [ ] MySQL 8.0+ database created via cPanel
- [ ] The `lume-deploy.zip` file (generated by `./deploy-cpanel.sh`)
- [ ] SSL certificate active on the domain (recommended)

### Generate the deployment package

On your local machine:

```bash
cd /path/to/lume
./deploy-cpanel.sh
```

This builds both frontends and produces `lume-deploy.zip` containing:

```
lume-deploy.zip
├── backend/          # Express.js API (source code)
├── frontend/            # Vue 3 SPA (source code + dist/ build)
│   └── dist/         # Pre-built static files ready to serve
│       └── .htaccess # SPA routing rules
├── demo/             # Nuxt 3 SSR (source code + .output/ build)
│   ├── .output/      # Pre-built SSR app
│   └── app.js        # Node.js entry point
└── README.md         # Quick reference
```

## Step 1: Create MySQL Database

### 1.1 Create the database

1. Go to **cPanel → MySQL Databases**
2. Create a new database (e.g., `riagri_lume`)
3. Create a new user (e.g., `riagri_admin`) with a strong password
4. Add the user to the database with **ALL PRIVILEGES**

> **Note:** cPanel prefixes database and user names with your account name.
> If your cPanel username is `riagri`, the full names will be `riagri_lume` and `riagri_admin`.

### 1.2 Note your credentials

You'll need these for the backend `.env` file:

```
Database name: riagri_lume
Database user: riagri_admin
Database password: (your password)
Database host: localhost
Database port: 3306
```

## Step 2: Deploy Backend API (Node.js App #1)

The backend is an Express.js API that serves data to both the admin panel and the public site.

### 2.1 Upload files

1. Unzip `lume-deploy.zip` on your local machine
2. Upload the entire `backend/` folder to `~/ri-agri.in/backend/` on the server
   - Use cPanel File Manager or SCP/SFTP

Your server should have:

```
~/ri-agri.in/backend/
├── src/              # Application source code
│   ├── index.js      # Entry point
│   ├── core/         # Framework core
│   └── modules/      # 23 pluggable modules
├── prisma/           # Database schema
├── package.json
├── uploads/          # File upload directory
└── logs/             # Application logs
```

### 2.2 Create Node.js application in cPanel

1. Go to **cPanel → Setup Node.js App**
2. Click **Create Application**
3. Fill in:

| Field | Value |
|-------|-------|
| Node.js version | `18` (or higher LTS) |
| Application mode | `Production` |
| Application root | `ri-agri.in/backend` |
| Application URL | `ri-agri.in/backend` |
| Application startup file | `src/index.js` |

4. Click **Create**
5. **Important:** Note the virtual environment activation command shown — you'll need it for the next steps

### 2.3 Install dependencies

Open **cPanel → Terminal** (or SSH into the server):

```bash
# Activate the Node.js virtual environment
# (use the exact command shown in cPanel after creating the app)
source /home/<cpanel-user>/nodevenv/ri-agri.in/backend/18/bin/activate

cd ~/ri-agri.in/backend
npm install --production
npx prisma generate
```

### 2.4 Configure environment variables

Create the `.env` file in `~/ri-agri.in/backend/`:

```env
# Server
NODE_ENV=production
PORT=3000
API_VERSION=v1

# Domain
BACKEND_URL=https://ri-agri.in/backend
FRONTEND_URL=https://ri-agri.in/frontend

# Database — use your cPanel MySQL credentials from Step 1
DATABASE_URL="mysql://riagri_admin:YOUR_PASSWORD@localhost:3306/riagri_lume"
DB_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=riagri_lume
DB_USER=riagri_admin
DB_PASSWORD=YOUR_PASSWORD
DB_POOL_SIZE=10
DB_LOGGING=false

# JWT — generate random strings (use: openssl rand -hex 32)
JWT_SECRET=GENERATE_A_RANDOM_64_CHAR_STRING
JWT_REFRESH_SECRET=GENERATE_ANOTHER_RANDOM_64_CHAR_STRING
JWT_EXPIRES_IN=7d

# CORS — allow admin panel and public site origins
CORS_ORIGIN=https://ri-agri.in

# Public website URL
PUBLIC_WEBSITE_URL=https://ri-agri.in/demo

# File uploads
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=10485760

# Email (configure with your SMTP provider)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
EMAIL_FROM=noreply@ri-agri.in

# Session
SESSION_SECRET=GENERATE_A_RANDOM_64_CHAR_STRING
```

> **Tip:** Generate random secrets with: `openssl rand -hex 32`

### 2.5 Initialize the database

```bash
cd ~/ri-agri.in/backend

# Create all database tables
npx prisma db push

# Seed roles, permissions, admin user, and default settings
npm run db:init
```

This creates:
- 6 roles (super_admin, admin, manager, staff, user, guest)
- 147+ permissions across all 23 modules
- Default admin account: `admin@lume.dev` / `admin123`
- Default application settings
- 6 page builder templates

### 2.6 Start the application

1. Go to **cPanel → Setup Node.js App**
2. Find the backend app → click **Restart**

### 2.7 Verify

```bash
curl https://ri-agri.in/backend/api/health
```

Expected response:
```json
{"success": true, "message": "OK"}
```

If this doesn't work, check `~/ri-agri.in/backend/logs/` for errors.

## Step 3: Deploy Admin Panel (Static Files — No Node.js App)

The admin panel is a pre-built Vue 3 Single Page Application. It consists entirely of static files (HTML, CSS, JS) — **no Node.js application is needed**.

### 3.1 Upload the built files

Upload the contents of `frontend/dist/` (**not** the `admin/` folder itself) to `~/ri-agri.in/frontend/`:

```
# What to upload:
frontend/dist/*  →  ~/ri-agri.in/frontend/

# Result on server:
~/ri-agri.in/frontend/
├── index.html        # SPA entry point
├── assets/           # JS, CSS bundles (hashed filenames)
│   ├── index-xxxxx.js
│   └── index-xxxxx.css
└── .htaccess         # SPA routing (IMPORTANT — hidden file!)
```

> **Important:** The `.htaccess` file is hidden by default. In cPanel File Manager, click **Settings** (top right) and check **Show Hidden Files** to verify it was uploaded.

### 3.2 Verify the `.htaccess` file

The `.htaccess` file handles SPA routing — without it, refreshing any page other than `/frontend/` will show a 404. It should contain:

```apache
RewriteEngine On
RewriteBase /frontend/

# ... (compression and caching rules) ...

RewriteCond %{REQUEST_FILENAME} -f [OR]
RewriteCond %{REQUEST_FILENAME} -l [OR]
RewriteCond %{REQUEST_FILENAME} -d
RewriteRule ^ - [L]

RewriteRule ^ /frontend/index.html [L]
```

### 3.3 Configure the API URL

The API URL is baked into the Vue build at build time. The deploy script builds with whatever `.env` / `.env.production` values exist in the frontend source.

If the admin panel can't connect to the API, you need to rebuild with the correct URL:

**Option A — Rebuild locally:**

Create `frontend/apps/web-lume/.env.production`:
```env
VITE_API_URL=https://ri-agri.in/backend/api
VITE_PUBLIC_SITE_URL=https://ri-agri.in/demo
```

Then rebuild and re-upload `dist/`:
```bash
cd frontend/apps/web-lume
npm run build
# Upload dist/* to server
```

**Option B — Use the source code on server:**

The `frontend/` folder in the zip also contains the full source code. You can rebuild on the server:
```bash
cd ~/ri-agri.in/frontend
npm install
# Edit .env.production with correct API URL
npm run build
# Move dist/* to the served directory
```

### 3.4 Verify

Open `https://ri-agri.in/frontend` in a browser.

Log in with the default admin credentials:

| Field | Value |
|-------|-------|
| Email | `admin@lume.dev` |
| Password | `admin123` |

> **Change the admin password immediately after first login.**

## Step 4: Deploy Public Website (Node.js App #2)

The public website is a Nuxt 3 server-side rendered app that fetches page content from the backend API and renders it on the server.

### 4.1 Upload files

Upload the entire `demo/` folder to `~/ri-agri.in/demo/` on the server.

Your server should have:

```
~/ri-agri.in/demo/
├── .output/          # Pre-built Nuxt SSR app
│   ├── server/       # Server bundle
│   │   └── index.mjs # Nitro server entry
│   └── public/       # Static assets
├── app.js            # cPanel entry point
├── package.json
├── pages/            # Source code (for rebuilds)
├── components/
├── composables/
├── layouts/
└── nuxt.config.ts
```

### 4.2 Create Node.js application in cPanel

1. Go to **cPanel → Setup Node.js App**
2. Click **Create Application**
3. Fill in:

| Field | Value |
|-------|-------|
| Node.js version | `18` (or higher LTS) |
| Application mode | `Production` |
| Application root | `ri-agri.in/demo` |
| Application URL | `ri-agri.in/demo` |
| Application startup file | `app.js` |

4. Click **Create**

### 4.3 Configure environment

Create `.env` in `~/ri-agri.in/demo/`:

```env
NUXT_PUBLIC_API_URL=https://ri-agri.in/backend/api
```

### 4.4 Install dependencies (optional)

The `.output/` directory contains the pre-built SSR app. If it's present and working, you may not need to install dependencies. However, if you need to rebuild:

```bash
source /home/<cpanel-user>/nodevenv/ri-agri.in/demo/18/bin/activate

cd ~/ri-agri.in/demo
npm install
npm run build
```

### 4.5 Start the application

1. Go to **cPanel → Setup Node.js App**
2. Find the demo app → click **Restart**

### 4.6 Verify

Open `https://ri-agri.in/demo` in a browser. You should see the public website with:
- Home page with hero section, stats, product categories, features
- Navigation menu with dropdown submenus
- All pages (Products, Services, About, Contact) rendering correctly

## Updating the Deployment

When you have code changes to deploy:

### Backend only
```bash
# Local: generate new zip
./deploy-cpanel.sh --skip-admin --skip-public

# Server: upload new backend/ files, then:
cd ~/ri-agri.in/backend
npm install --production
npx prisma generate
npx prisma db push    # if schema changed
# Restart Node.js app in cPanel
```

### Admin panel only
```bash
# Local: rebuild
cd frontend/apps/web-lume
npm run build

# Server: upload dist/* to ~/ri-agri.in/frontend/
# No restart needed — static files update immediately
```

### Public site only
```bash
# Local: generate new zip
./deploy-cpanel.sh --skip-admin --skip-backend

# Server: upload new demo/ files, then:
# Restart Node.js app in cPanel
```

### Full redeployment
```bash
# Local
./deploy-cpanel.sh

# Server: upload all three folders, restart both Node.js apps
```

## Troubleshooting

### Backend returns 502 or 503

- Check the Node.js app is running in **cPanel → Node.js Apps**
- Check logs: `~/ri-agri.in/backend/logs/error.log`
- Verify `.env` has correct `DATABASE_URL`
- Try restarting the app

### Admin panel shows blank white page

- Verify `index.html` exists in `~/ri-agri.in/frontend/`
- Verify `.htaccess` is present (enable "Show Hidden Files" in File Manager)
- Check browser console (F12) for errors — likely API connection issue
- Ensure `VITE_API_URL` was set correctly before building

### Admin panel returns 404 on page refresh

- The `.htaccess` file is missing or not working
- Verify `mod_rewrite` is enabled (it is by default on most cPanel hosts)
- Check `RewriteBase` is set to `/frontend/`

### Public site shows "Cannot connect" or 500 error

- Verify the Node.js app is running for `/demo`
- Verify `app.js` exists and `.output/` directory is present
- Check `.env` has `NUXT_PUBLIC_API_URL` pointing to the backend
- Verify the backend is running and accessible

### CORS errors in browser console

- Update `CORS_ORIGIN` in backend `.env` to include the exact origin
- Must be `https://ri-agri.in` (no trailing slash, correct protocol)
- Restart the backend Node.js app after changing `.env`

### Database connection refused

- Verify MySQL credentials in **cPanel → MySQL Databases**
- Remember cPanel prefixes: `cpaneluser_dbname`, `cpaneluser_dbuser`
- Test connection: `mysql -u riagri_admin -p riagri_lume`
- Check `DATABASE_URL` format: `mysql://user:password@localhost:3306/dbname`

### "Permission denied" errors

- File permissions should be `644` for files, `755` for directories
- The `uploads/` directory needs write permission: `chmod 755 ~/ri-agri.in/backend/uploads`

### Node.js app won't start

- Check the startup file path is correct in cPanel Node.js settings
- Backend: `src/index.js` (not `index.js`)
- Public site: `app.js` (not `server/index.mjs`)
- Run manually to see errors:
  ```bash
  cd ~/ri-agri.in/backend
  source /home/<user>/nodevenv/ri-agri.in/backend/18/bin/activate
  node src/index.js
  ```
